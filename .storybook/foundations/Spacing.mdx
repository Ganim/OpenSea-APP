# Espaçamento

A escala de espaçamento segue o **default do Tailwind 4**, baseado em uma unidade
de `0.25rem` (4px) por _step_. O OpenSea-APP **não sobrescreve** essa escala via
`@theme` em `globals.css` — o que está pintado abaixo é exatamente o que o framework
fornece.

Use as classes utilitárias `p-{n}`, `m-{n}`, `gap-{n}`, `space-x-{n}`, `space-y-{n}`,
`px-{n}` etc. com os valores documentados aqui.

## Escala canônica (0–16)

Cada barra azul abaixo representa o tamanho real em pixels. Steps acima de 16
(20, 24, 32, 40, 48, 56, 64, ...) seguem o mesmo princípio: `step * 0.25rem`.

## Onde isto vem

- **Fonte de verdade:** default do Tailwind 4 (não há `--spacing-*` custom em
  `@theme` no `globals.css`).
- **Como mudar:** adicionar `--spacing` ou tokens próprios dentro do bloco
  `@theme {}` em `src/app/globals.css`. Atualmente o bloco só sobrescreve cores
  e a sidebar — espaçamento permanece padrão.
- **Boas práticas:**
  - Cards de detalhe usam `p-5` (CLAUDE.md §9 — Identity Card).
  - Cards de form usam `py-2` reduzido (CLAUDE.md §9 — Form Card).
  - `Tabs` headers ganham `mb-4` quando têm `TabsList` (CLAUDE.md §9).
  - `gap-2` / `gap-3` em ações inline; `gap-4` / `gap-6` em seções de form.

> Para conferir um valor desconhecido: `step * 4 = px`. Ex.: `p-7` → 28px.
