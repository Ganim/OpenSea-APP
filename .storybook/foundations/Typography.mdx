# Tipografia

A escala tipográfica segue o **default do Tailwind 4** (mesmo princípio da escala
de espaçamento). A família principal é a do Next.js / `next/font` — tipicamente
**Geist Sans** para texto e **Geist Mono** para código, configuradas em
`src/app/layout.tsx`. Se for adicionar outra família custom, faça via `@theme {}`
em `src/app/globals.css`.

## Família de fonte

- **Sans (default):** Geist Sans via `next/font/google` em `src/app/layout.tsx`
  (variável CSS `--font-geist-sans`). Tailwind aplica via `font-sans` (utilitário
  default).
- **Mono:** Geist Mono via `next/font/google` (variável `--font-geist-mono`,
  utilitário `font-mono`).
- Para aplicar pesos, use as utilitárias do Tailwind: `font-medium` (500),
  `font-semibold` (600), `font-bold` (700). Pesos abaixo de 400 são raros no
  produto.

## Pesos recomendados

| Peso | Uso                                          |
| ---- | -------------------------------------------- |
| 400  | body padrão                                  |
| 500  | labels, navegação, CTAs secundários          |
| 600  | títulos de card, subtítulos, valores em KPIs |
| 700  | títulos de página, números de destaque       |

## Lembrete PT-BR

Toda string visível ao usuário (label, placeholder, toast, título, mensagem de
erro, modal, breadcrumb) é em **português formal com acentuação correta**
(CLAUDE.md §10). Logs, comentários e identificadores em código permanecem em
inglês. Se você está revisando uma tela e encontra "Save", "Cancel", "Loading…"
em string visível, é bug.
